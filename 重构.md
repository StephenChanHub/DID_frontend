## 一、 环境与基础配置

### 1. 基础信息

- **后端 API 地址**: `http://192.168.64.2:3000/api`
- **技术栈**: Vue 3 (Composition API), Axios, Pinia, localStorage.

### 2. Axios 实例配置 (`src/utils/request.ts`)

需配置全局拦截器，确保每次请求自动携带 **JWT Token**。

```typescript
import axios from "axios";

const request = axios.create({
  baseURL: "http://192.168.64.2:3000/api", //
  timeout: 5000,
});

// 请求拦截器：注入 Token
request.interceptors.request.use((config) => {
  const token = localStorage.getItem("did_token");
  if (token) {
    config.headers.Authorization = `Bearer ${token}`; // 对应后端 authenticateToken
  }
  return config;
});

export default request;
```

## 二、 核心 API 接口说明

### 1. 发送邮箱验证码

- **路径**: `POST /auth/send-code`
- **功能**: 后端生成 6 位码存入 Redis 并通过 QQ 邮箱发送。
- **请求体**:
  ```json
  { "email": "stephen@example.com" }
  ```
- **响应示例**:
  ```json
  { "message": "验证码已发送，请检查邮箱" }
  ```

### 2. 用户注册

- **路径**: `POST /auth/register`
- **功能**: 校验 Redis 验证码，哈希密码并存入 MySQL。
- **请求体**:
  ```json
  {
    "email": "stephen@example.com",
    "password": "your_password",
    "code": "123456",
    "nickname": "Stephen"
  }
  ```
- **响应示例**:
  ```json
  {
    "message": "注册成功",
    "token": "eyJhbG...",
    "userId": 1
  }
  ```

### 3. 用户登录

- **路径**: `POST /auth/login`
- **请求体**:
  ```json
  { "email": "stephen@example.com", "password": "your_password" }
  ```
- **响应示例**:
  ```json
  {
    "message": "登录成功",
    "token": "eyJhbG...",
    "user": { "nickname": "Stephen", "level": "A1" }
  }
  ```

## 三、 前端功能实现指南 (给 Claude 的指令)

请要求 Claude 按照以下逻辑重构 `AuthModal.vue`：

### 1. 状态管理 (Pinia Store)

创建一个 `userStore`，用于存储当前登录用户的 `token` 和基本信息，并同步到 `localStorage`。

### 2. 验证码交互逻辑

- **倒计时控制**: 点击“发送验证码”后，按钮进入 `disabled` 状态并开始 **60 秒倒计时**。
- **频率限制**: 倒计时结束前禁止重复调用接口。

### 3. 注册/登录切换

- 模态框支持 **Login** 和 **Register** 两种模式切换。
- 注册模式下显示 `nickname` 和 `code` (验证码) 输入框；登录模式下隐藏。

### 4. 登录后的 UI 响应

- **持久化**: 拿到 `token` 后立即存入 `localStorage`。
- **状态更新**: 成功后关闭 Modal，导航栏从“登录/注册”变为展示“用户昵称”。

## 四、 核心业务逻辑流图

### 💡 关键注意事项

1.  **跨域处理**: 如果前端运行在 `localhost:5173`，需在 Vite 的 `vite.config.ts` 中配置 `server.proxy` 指向 `192.168.64.2:3000`。
2.  **错误处理**: 后端返回 `400`（验证码错误）或 `404`（用户不存在）时，前端需通过 `ElMessage` 或 `alert` 友好提示。
3.  **安全性**: 即使前端有校验，后端已实现 `bcrypt` 加密和 `JWT` 签名，确保数据在传输和存储过程中的安全。
