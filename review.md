#### 第一层：整体架构（The Big Picture）
> “我的项目采用了经典的 **前后端分离** 架构。为了模拟真实的生产环境，我将后端部署在 **UTM 虚拟化的 Ubuntu** 系统中，前端在 Mac 宿主机运行。”

#### 第二层：前端技术栈（Frontend Stack）
> “前端基于 **Vue 3** 开发，并引入了 **TypeScript** 增强代码的严谨性。
> * 使用 **Vite** 作为构建工具，确保极速的热更新。
> * 状态管理使用 **Pinia**，用来实时同步用户的**积分**、**等级**以及等级是否因过期而**灰度显示**的状态。”

#### 第三层：后端与运行环境（Backend & Runtime）
> “后端运行在 **Node.js** 环境下，框架选择了 **Express**。
> * 全程使用 **TypeScript**，实现前后端类型定义的统一。
> * 核心逻辑通过 RESTful API 提供支持，并使用 **CORS** 解决跨设备通信的跨域问题。”

#### 第四层：数据持久化（Data Layer）
> “数据库选用 **MySQL**。
> * 我设计了一套精简的表结构，通过 **JSON 字段** 灵活处理选择题选项。
> * 用户安全方面，不存储明文密码，而是通过 **Bcrypt** 加密存储，并配合 **JWT** 进行身份校验。”

---

### 重点
1.  **Restful API：** 前后端沟通规范
2.  **Stateless（无状态）：** 配合 JWT 服务器不存储 Session，扩展性好。

### 总结

* **底座：** Node.js + Ubuntu (UTM)
* **语言：** TypeScript (贯穿前后端)
* **后端：** Express + MySQL
* **前端：** Vue 3 + Pinia + Axios
